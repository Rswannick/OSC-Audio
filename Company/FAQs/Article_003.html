Logic Pro X - AU Validation [RESOLVED]

We were made aware of some users having some issues with AU plug-in validation in Logic Pro X. We've resolved this behavior and the steps below will get you working.

  1. Open a Finder window and using the top navigation bar open the Go tab.
  2. Hold the option key on your keyboard and select the newly visible (user hidden) Library.
  3. Inside this Library find the Caches folder and inside this, the AudioUnitCache folder.
  4. Finder and delete all files in this folder.
  5. Close and reopen Logic Pro X* and you're good to go!
  6. If your app does not immediately appear, use the Settings > Plug-in Manager to rescan.


    7. *If you are on an Apple Silicon based machine, disable Rosetta for the Logic Pro X app by right-clicking it's icon and selecting Get Info.



    If you continue to have issues. Delete sumFX using the included uninstaller, or manually removing your .component file from your Library > Audio > Plug-ins > Component folder. Restart your system before reinstalling the latest app version.



If none of these items help or you're stuck, contact our team at the link below and be sure to include the text from the Logic Pro X plug-in scanning tool's window.
